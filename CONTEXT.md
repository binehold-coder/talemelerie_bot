## Цель

Прием заказов на выпечку через Telegram Mini App, хранение в Google Sheets, уведомления о готовности.

## Стек

* Бекенд: aiogram 3.x, Python 3.12
* Хранилище: Google Sheets (gspread)
* Фронтенд: HTML/JS (Telegram WebApp), Vercel
* Деплой: IONOS (VPS)

## Роли

* **Муж** — разработчик, архитектор, отвечает за код и деплой.
* **Жена** — респонсабль магазина, будет договариваться о внедрении бота с директором сети.

## Текущее состояние (УЖЕ РАБОТАЕТ)

* Бот принимает заказы через WebApp с валидацией (имя, телефон, дата/время, товары).
* Автоматический пересчет total, если фронт не прислал.
* Заказы пишутся в Google Sheets.
* Генерация Order ID: `LT-{BAKERY_CODE}-{YYYY}-{MM}-{NNN}` (месяц первым для удобного поиска, хронология — по колонке Date Commande).
* Счетчики в технической вкладке `Order Counters` (колонки: `Prefix`, `Last sequence`), скрытой и защищенной от редактирования.
* Напоминания о pickup за 1 минуту.
* Защита от дублей (5 минут между заказами с одного chat_id).
* Ретраи (3 попытки с экспоненциальной задержкой) при записи в Google Sheets.
* Миграция схемы Google Sheets с бэкапом.

## Ближайшие задачи (ПО ПОРЯДКУ)

### 1. Переименовать вкладки и настроить списки в Google Sheets

* `Лист 1` → `Commandes` (не сортируется вручную)
* `Order Counters` → `Compteurs`
* Создать новую вкладку `Config` (скрытая, для списка статусов)
* Настроить Data Validation для колонки `Statut` (выпадающий список, предпочтительнее скриптов):
* `Nouveau` — жёлтый
* `En préparation` — оранжевый
* `Prêt` — зелёный
* `Retiré` — серый
* `Annulé` — красный



### 2. Вкладка Production

* Авто-фильтр активных заказов.
* Показывает только незакрытые заказы со статусами: Nouveau, En préparation, Prêt.
* Исключает: Retiré, Annulé.

### 3. Вкладка Dashboard (аналитика)

Автоматическая аналитика (через формулу QUERY или сводную таблицу):

* Количество заказов за день/неделю/месяц
* Общая сумма (все заказы)
* Фактическая выручка (только Retiré)
* Отменённые заказы и сумма
* Средний чек
* Статусы в работе (количество по каждому)
* Топ-5 популярных товаров
* Распределение по часам самовывоза

### 3.5. Ограничение срока заказа (60 дней)

* Фронтенд: календарь блокирует выбор даты > 60 дней от сегодня.
* Бекенд: при получении заказа проверяет, что pickup_time <= today + 60 дней.
* Если дата позже — возвращать ошибку с сообщением и номером телефона из .env.
* Сообщение: «Pour les commandes au-delà de 60 jours, veuillez nous contacter directement : 📞 +33 X XX XX XX XX»
* Часы работы для самовывоза:
  - Понедельник — Суббота: 06:30 – 19:30
  - Воскресенье: 06:30 – 13:00
* При выборе времени в WebApp календарь должен учитывать эти ограничения.
* Бекенд также проверяет pickup_time и возвращает ошибку, если время вне допустимого диапазона.

### 4. Отмена заказа (для клиента)

* Доступно только для статуса `Nouveau`.
* Если статус выше (например, `En préparation` или выше) — показывать номер булочной из `.env` (`BAKERY_PHONE`), чтобы текст с номером автоматически становился кликабельным для звонка:

> Votre commande ne peut plus être annulée автоматически, car sa préparation peut déjà avoir commencé.
> Pour toute demande, appelez-nous :
> 📞 +33 X XX XX XX XX

### 5. Подтверждение получения (Retiré)

Сценарий с защитой от случайного нажатия:

1. Продавец ставит статус `Prêt`.
2. Бот отправляет клиенту: *«Votre commande est prête»* + кнопка `✅ J’ai récupéré ma commande`.
3. После нажатия — повторное двойное подтверждение: *«Confirmez-vous avoir récupéré votre commande ?»* с кнопками `Oui, confirmer` / `Pas encore`.
4. Только при нажатии `Oui, confirmer` статус меняется на `Retiré`.

### 6. Тесты

* Юнит-тесты для валидации (`order.py`).
* Интеграционные тесты для записи в Google Sheets (с мок-объектами).

### 7. Переход на PostgreSQL (при расширении до 3+ булочных)

* Заменить Google Sheets на PostgreSQL.
* Переписать блокировку и счетчики на атомарные `UPDATE ... RETURNING`.

## Ключевые ограничения и решения

* Нагрузка: 30-40 заказов/месяц (масштабирование на старте не требуется).
* Часовой пояс: `Europe/Paris` (пользователи локальные, UTC не используем).
* Блокировка: `asyncio.Lock` (один инстанс бота).
* Секреты: хранятся в `.env` и `credentials.json` (не в коде).
* Номер булочной: из `.env` (`BAKERY_PHONE`), без хардкода.
* Язык интерфейса: французский.
* Вся команда работает с Google Sheets как на мобильных устройствах, так и на десктопе.
